Postman 学習ノート

第1週：Postmanの基礎理解

APIとは「アプリケーション同士のやりとりの窓口」のことで、Postmanはそのやりとりを可視化・テストできるツールです。
第1週では以下のことを学習しました：

  • PostmanのインストールとUI操作
  • リクエスト（GET/POST）とは何か
  • https://jsonplaceholder.typicode.com や worldtimeapi.org などのモックAPIを使ったリクエスト
  • レスポンスの形式（JSON）とその読み方
  • クエリパラメータ・ヘッダー・ボディの違い

→ 「こういうデータをください」とサーバーにお願いする感覚を掴みました。

第2週：レスポンス検証とテストスクリプト入門

APIから返ってくるレスポンスが「正しいか」を自動チェックする方法を学びました。これにより、バグを早期に発見できるようになります。

  • pm.test() でテスト名を定義
  • pm.expect() で期待する値を定義
  • ステータスコード（例: 200 OK）の検証
  • JSONレスポンスのキーの有無・値の検証

例： 

pm.test("ステータスコードは200", function () {
  pm.response.to.have.status(200);
});

pm.test("1番目のユーザー名がLeanne Graham", function () {
  const json = pm.response.json();
  pm.expect(json[0].name).to.eql("Leanne Graham");
});

→ Postmanでテストを書くことにより、実務でのAPI検証や不具合の早期発見ができるようになる準備が整いました。
